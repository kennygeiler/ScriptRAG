# Plan 08-01 — Data out tab & export surface (OUT-01)

## Requirement

**OUT-01:** After HITL load, operators can see **schema documentation**, run **fixed recipe Cypher** (no NL), and **download CSV** (narrative edges, characters, events) from Streamlit — proving manipulable data without extra tooling.

## Scope

- New module `data_out.py`: parameterized Cypher only; reuses `metrics.NARRATIVE_REL_TYPES`.
- New tab **Data out** in `app.py` after **Reconcile**, before **Pipeline Efficiency Tracking**.
- Cached reads keyed on existing `_neo4j_dashboard_cache_stamp()` pattern.
- Reconcile tab caption prefixed as **optional post-load hygiene** (narrative alignment).

## Out of scope (later plans)

- Guided wizard / CEO mode (Phase 10).
- Cleanup progress strip (Phase 9).
- JSON bundle export of full graph from UI (optional follow-up).

## Verification

1. With an empty Neo4j, **Data out** shows informative empty state (no traceback).
2. After load, label/rel counts populate; each recipe query runs without error.
3. CSV downloads produce non-empty files when graph has data; narrative edge export respects row limit.
4. `uv run python -c "import data_out"` succeeds.
